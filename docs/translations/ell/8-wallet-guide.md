## Επισκόπηση

### Χαρακτηριστικά Πορτοφολιού Bitcoin-PoCX

Το Qt πορτοφόλι Bitcoin-PoCX (`bitcoin-qt`) παρέχει:
- Τυπική λειτουργικότητα πορτοφολιού Bitcoin Core (αποστολή, λήψη, διαχείριση συναλλαγών)
- **Διαχειριστής Εκχωρήσεων Σφυρηλάτησης**: Γραφικό περιβάλλον για δημιουργία/ανάκληση εκχωρήσεων plot
- **Λειτουργία Διακομιστή Εξόρυξης**: Η σημαία `-miningserver` ενεργοποιεί λειτουργίες σχετικές με την εξόρυξη
- **Ιστορικό Συναλλαγών**: Εμφάνιση συναλλαγών εκχώρησης και ανάκλησης

### Εκκίνηση του Πορτοφολιού

**Μόνο Κόμβος** (χωρίς εξόρυξη):
```bash
./build/bin/bitcoin-qt
```

**Με Εξόρυξη** (ενεργοποιεί τον διάλογο εκχωρήσεων):
```bash
./build/bin/bitcoin-qt -server -miningserver
```

**Εναλλακτική Γραμμής Εντολών**:
```bash
./build/bin/bitcoind -miningserver
```

### Απαιτήσεις Εξόρυξης

**Για Λειτουργίες Εξόρυξης**:
- Απαιτείται η σημαία `-miningserver`
- Πορτοφόλι με διευθύνσεις P2WPKH και ιδιωτικά κλειδιά
- Εξωτερικό εργαλείο δημιουργίας plot (`pocx_plotter`) για παραγωγή plot
- Εξωτερικός miner (`pocx_miner`) για εξόρυξη

**Για Εξόρυξη σε Pool**:
- Δημιουργία εκχώρησης σφυρηλάτησης στη διεύθυνση του pool
- Δεν απαιτείται πορτοφόλι στον διακομιστή του pool (το pool διαχειρίζεται τα κλειδιά)

## Διάλογος Εκχώρησης Σφυρηλάτησης

### Πρόσβαση στον Διάλογο

**Μενού**: `Πορτοφόλι → Εκχωρήσεις Σφυρηλάτησης`
**Γραμμή Εργαλείων**: Εικονίδιο εξόρυξης (ορατό μόνο με τη σημαία `-miningserver`)
**Μέγεθος Παραθύρου**: 600×450 pixels

### Λειτουργίες Διαλόγου

#### Λειτουργία 1: Δημιουργία Εκχώρησης

**Σκοπός**: Ανάθεση δικαιωμάτων σφυρηλάτησης σε pool ή άλλη διεύθυνση διατηρώντας την ιδιοκτησία του plot.

**Περιπτώσεις Χρήσης**:
- Εξόρυξη σε pool (εκχώρηση στη διεύθυνση του pool)
- Ψυχρή αποθήκευση (κλειδί εξόρυξης ξεχωριστό από την ιδιοκτησία plot)
- Κοινόχρηστη υποδομή (ανάθεση σε θερμό πορτοφόλι)

**Απαιτήσεις**:
- Διεύθυνση plot (P2WPKH bech32, πρέπει να κατέχετε το ιδιωτικό κλειδί)
- Διεύθυνση σφυρηλάτησης (P2WPKH bech32, διαφορετική από τη διεύθυνση plot)
- Ξεκλείδωτο πορτοφόλι (αν είναι κρυπτογραφημένο)
- Η διεύθυνση plot έχει επιβεβαιωμένα UTXOs

**Βήματα**:
1. Επιλέξτε τη λειτουργία "Δημιουργία Εκχώρησης"
2. Επιλέξτε διεύθυνση plot από την αναπτυσσόμενη λίστα ή εισάγετε χειροκίνητα
3. Εισάγετε τη διεύθυνση σφυρηλάτησης (pool ή εκπρόσωπος)
4. Κάντε κλικ στο "Αποστολή Εκχώρησης" (το κουμπί ενεργοποιείται όταν οι είσοδοι είναι έγκυρες)
5. Η συναλλαγή εκπέμπεται αμέσως
6. Η εκχώρηση ενεργοποιείται μετά από `nForgingAssignmentDelay` blocks:
   - Mainnet/Testnet: 30 blocks (~1 ώρα)
   - Regtest: 4 blocks (~4 δευτερόλεπτα)

**Τέλος Συναλλαγής**: Προεπιλογή 10× `minRelayFee` (προσαρμόσιμο)

**Δομή Συναλλαγής**:
- Είσοδος: UTXO από τη διεύθυνση plot (αποδεικνύει την ιδιοκτησία)
- Έξοδος OP_RETURN: Δείκτης `POCX` + διεύθυνση_plot + διεύθυνση_σφυρηλάτησης (46 bytes)
- Έξοδος ρέστων: Επιστρέφεται στο πορτοφόλι

#### Λειτουργία 2: Ανάκληση Εκχώρησης

**Σκοπός**: Ακύρωση εκχώρησης σφυρηλάτησης και επιστροφή δικαιωμάτων στον ιδιοκτήτη του plot.

**Απαιτήσεις**:
- Διεύθυνση plot (πρέπει να κατέχετε το ιδιωτικό κλειδί)
- Ξεκλείδωτο πορτοφόλι (αν είναι κρυπτογραφημένο)
- Η διεύθυνση plot έχει επιβεβαιωμένα UTXOs

**Βήματα**:
1. Επιλέξτε τη λειτουργία "Ανάκληση Εκχώρησης"
2. Επιλέξτε τη διεύθυνση plot
3. Κάντε κλικ στο "Αποστολή Ανάκλησης"
4. Η συναλλαγή εκπέμπεται αμέσως
5. Η ανάκληση τίθεται σε ισχύ μετά από `nForgingRevocationDelay` blocks:
   - Mainnet/Testnet: 720 blocks (~24 ώρες)
   - Regtest: 8 blocks (~8 δευτερόλεπτα)

**Αποτέλεσμα**:
- Η διεύθυνση σφυρηλάτησης μπορεί ακόμα να σφυρηλατεί κατά τη διάρκεια της καθυστέρησης
- Ο ιδιοκτήτης του plot ανακτά τα δικαιώματα μετά την ολοκλήρωση της ανάκλησης
- Μπορεί να δημιουργηθεί νέα εκχώρηση στη συνέχεια

**Δομή Συναλλαγής**:
- Είσοδος: UTXO από τη διεύθυνση plot (αποδεικνύει την ιδιοκτησία)
- Έξοδος OP_RETURN: Δείκτης `XCOP` + διεύθυνση_plot (26 bytes)
- Έξοδος ρέστων: Επιστρέφεται στο πορτοφόλι

#### Λειτουργία 3: Έλεγχος Κατάστασης Εκχώρησης

**Σκοπός**: Ερώτηση της τρέχουσας κατάστασης εκχώρησης για οποιαδήποτε διεύθυνση plot.

**Απαιτήσεις**: Καμία (μόνο ανάγνωση, δεν χρειάζεται πορτοφόλι)

**Βήματα**:
1. Επιλέξτε τη λειτουργία "Έλεγχος Κατάστασης Εκχώρησης"
2. Εισάγετε τη διεύθυνση plot
3. Κάντε κλικ στο "Έλεγχος Κατάστασης"
4. Το πλαίσιο κατάστασης εμφανίζει την τρέχουσα κατάσταση με λεπτομέρειες

**Ενδείξεις Κατάστασης** (με χρωματική κωδικοποίηση):

**Γκρι - ΧΩΡΙΣ ΕΚΧΩΡΗΣΗ**
```
UNASSIGNED - Δεν υπάρχει εκχώρηση
```

**Πορτοκαλί - ΣΕ ΕΚΧΩΡΗΣΗ**
```
ASSIGNING - Η εκχώρηση εκκρεμεί ενεργοποίηση
Διεύθυνση Σφυρηλάτησης: pocx1qforger...
Δημιουργήθηκε στο ύψος: 12000
Ενεργοποιείται στο ύψος: 12030 (απομένουν 5 blocks)
```

**Πράσινο - ΕΚΧΩΡΗΜΕΝΟ**
```
ASSIGNED - Ενεργή εκχώρηση
Διεύθυνση Σφυρηλάτησης: pocx1qforger...
Δημιουργήθηκε στο ύψος: 12000
Ενεργοποιήθηκε στο ύψος: 12030
```

**Κόκκινο-Πορτοκαλί - ΣΕ ΑΝΑΚΛΗΣΗ**
```
REVOKING - Η ανάκληση εκκρεμεί
Διεύθυνση Σφυρηλάτησης: pocx1qforger... (ακόμα ενεργή)
Η εκχώρηση δημιουργήθηκε στο ύψος: 12000
Ανακλήθηκε στο ύψος: 12300
Η ανάκληση τίθεται σε ισχύ στο ύψος: 13020 (απομένουν 50 blocks)
```

**Κόκκινο - ΑΝΑΚΛΗΜΕΝΟ**
```
REVOKED - Η εκχώρηση ανακλήθηκε
Προηγουμένως εκχωρημένο σε: pocx1qforger...
Η εκχώρηση δημιουργήθηκε στο ύψος: 12000
Ανακλήθηκε στο ύψος: 12300
Η ανάκληση τέθηκε σε ισχύ στο ύψος: 13020
```

## Ιστορικό Συναλλαγών

### Εμφάνιση Συναλλαγής Εκχώρησης

**Τύπος**: "Εκχώρηση"
**Εικονίδιο**: Εικονίδιο εξόρυξης (ίδιο με τα εξορυγμένα blocks)

**Στήλη Διεύθυνσης**: Διεύθυνση plot (διεύθυνση της οποίας τα δικαιώματα σφυρηλάτησης εκχωρούνται)
**Στήλη Ποσού**: Τέλος συναλλαγής (αρνητικό, εξερχόμενη συναλλαγή)
**Στήλη Κατάστασης**: Αριθμός επιβεβαιώσεων (0-6+)

**Λεπτομέρειες** (όταν γίνεται κλικ):
- Αναγνωριστικό Συναλλαγής
- Διεύθυνση plot
- Διεύθυνση σφυρηλάτησης (αναλυμένη από OP_RETURN)
- Δημιουργήθηκε στο ύψος
- Ύψος ενεργοποίησης
- Τέλος συναλλαγής
- Χρονοσήμανση

### Εμφάνιση Συναλλαγής Ανάκλησης

**Τύπος**: "Ανάκληση"
**Εικονίδιο**: Εικονίδιο εξόρυξης

**Στήλη Διεύθυνσης**: Διεύθυνση plot
**Στήλη Ποσού**: Τέλος συναλλαγής (αρνητικό)
**Στήλη Κατάστασης**: Αριθμός επιβεβαιώσεων

**Λεπτομέρειες** (όταν γίνεται κλικ):
- Αναγνωριστικό Συναλλαγής
- Διεύθυνση plot
- Ανακλήθηκε στο ύψος
- Ύψος ισχύος ανάκλησης
- Τέλος συναλλαγής
- Χρονοσήμανση

### Φιλτράρισμα Συναλλαγών

**Διαθέσιμα Φίλτρα**:
- "Όλα" (προεπιλογή, περιλαμβάνει εκχωρήσεις/ανακλήσεις)
- Εύρος ημερομηνιών
- Εύρος ποσών
- Αναζήτηση με διεύθυνση
- Αναζήτηση με αναγνωριστικό συναλλαγής
- Αναζήτηση με ετικέτα (αν η διεύθυνση έχει ετικέτα)

**Σημείωση**: Οι συναλλαγές Εκχώρησης/Ανάκλησης εμφανίζονται προς το παρόν στο φίλτρο "Όλα". Δεν έχει υλοποιηθεί ακόμη αποκλειστικό φίλτρο τύπου.

### Ταξινόμηση Συναλλαγών

**Σειρά Ταξινόμησης** (ανά τύπο):
- Παραγόμενες (τύπος 0)
- Ληφθείσες (τύπος 1-3)
- Εκχώρηση (τύπος 4)
- Ανάκληση (τύπος 5)
- Απεσταλμένες (τύπος 6+)

## Απαιτήσεις Διευθύνσεων

### Μόνο P2WPKH (SegWit v0)

**Οι λειτουργίες σφυρηλάτησης απαιτούν**:
- Διευθύνσεις κωδικοποιημένες σε Bech32 (που αρχίζουν με "pocx1q" mainnet, "tpocx1q" testnet, "rpocx1q" regtest)
- Μορφή P2WPKH (Pay-to-Witness-Public-Key-Hash)
- Hash κλειδιού 20 bytes

**ΔΕΝ Υποστηρίζονται**:
- P2PKH (παλαιού τύπου, που αρχίζουν με "1")
- P2SH (wrapped SegWit, που αρχίζουν με "3")
- P2TR (Taproot, που αρχίζουν με "bc1p")

**Αιτιολογία**: Οι υπογραφές block PoCX απαιτούν συγκεκριμένη μορφή witness v0 για την επικύρωση αποδείξεων.

### Φιλτράρισμα Αναπτυσσόμενης Λίστας Διευθύνσεων

**ComboBox Διεύθυνσης Plot**:
- Συμπληρώνεται αυτόματα με τις διευθύνσεις λήψης του πορτοφολιού
- Φιλτράρει τις μη-P2WPKH διευθύνσεις
- Εμφανίζει τη μορφή: "Ετικέτα (διεύθυνση)" αν έχει ετικέτα, διαφορετικά μόνο τη διεύθυνση
- Πρώτο στοιχείο: "-- Εισάγετε προσαρμοσμένη διεύθυνση --" για χειροκίνητη εισαγωγή

**Χειροκίνητη Εισαγωγή**:
- Επικυρώνει τη μορφή κατά την εισαγωγή
- Πρέπει να είναι έγκυρο bech32 P2WPKH
- Το κουμπί απενεργοποιείται αν η μορφή είναι μη έγκυρη

### Μηνύματα Σφάλματος Επικύρωσης

**Σφάλματα Διαλόγου**:
- "Η διεύθυνση plot πρέπει να είναι P2WPKH (bech32)"
- "Η διεύθυνση σφυρηλάτησης πρέπει να είναι P2WPKH (bech32)"
- "Μη έγκυρη μορφή διεύθυνσης"
- "Δεν υπάρχουν διαθέσιμα κέρματα στη διεύθυνση plot. Δεν είναι δυνατή η απόδειξη ιδιοκτησίας."
- "Δεν είναι δυνατή η δημιουργία συναλλαγών με πορτοφόλι μόνο παρακολούθησης"
- "Το πορτοφόλι δεν είναι διαθέσιμο"
- "Το πορτοφόλι είναι κλειδωμένο" (από RPC)

## Ενσωμάτωση Εξόρυξης

### Απαιτήσεις Ρύθμισης

**Διαμόρφωση Κόμβου**:
```bash
# bitcoin.conf
miningserver=1
server=1
```

**Απαιτήσεις Πορτοφολιού**:
- Διευθύνσεις P2WPKH για ιδιοκτησία plot
- Ιδιωτικά κλειδιά για εξόρυξη (ή διεύθυνση σφυρηλάτησης αν χρησιμοποιούνται εκχωρήσεις)
- Επιβεβαιωμένα UTXOs για δημιουργία συναλλαγών

**Εξωτερικά Εργαλεία**:
- `pocx_plotter`: Παραγωγή αρχείων plot
- `pocx_miner`: Σάρωση plots και υποβολή nonces

### Ροή Εργασίας

#### Εξόρυξη Solo

1. **Παραγωγή Αρχείων Plot**:
   ```bash
   pocx_plotter --account <hash160_διεύθυνσης_plot> --seed <32_bytes> --nonces <πλήθος>
   ```

2. **Εκκίνηση Κόμβου** με διακομιστή εξόρυξης:
   ```bash
   bitcoin-qt -server -miningserver
   ```

3. **Διαμόρφωση Miner**:
   - Κατεύθυνση στο RPC endpoint του κόμβου
   - Καθορισμός καταλόγων αρχείων plot
   - Διαμόρφωση αναγνωριστικού λογαριασμού (από τη διεύθυνση plot)

4. **Έναρξη Εξόρυξης**:
   ```bash
   pocx_miner --rpc-url http://localhost:8332 --plots /διαδρομή/προς/plots
   ```

5. **Παρακολούθηση**:
   - Ο miner καλεί `get_mining_info` σε κάθε block
   - Σαρώνει plots για την καλύτερη προθεσμία
   - Καλεί `submit_nonce` όταν βρεθεί λύση
   - Ο κόμβος επικυρώνει και σφυρηλατεί το block αυτόματα

#### Εξόρυξη σε Pool

1. **Παραγωγή Αρχείων Plot** (ίδια με την εξόρυξη solo)

2. **Δημιουργία Εκχώρησης Σφυρηλάτησης**:
   - Ανοίξτε τον Διάλογο Εκχώρησης Σφυρηλάτησης
   - Επιλέξτε τη διεύθυνση plot
   - Εισάγετε τη διεύθυνση σφυρηλάτησης του pool
   - Κάντε κλικ στο "Αποστολή Εκχώρησης"
   - Περιμένετε την καθυστέρηση ενεργοποίησης (30 blocks testnet)

3. **Διαμόρφωση Miner**:
   - Κατεύθυνση στο endpoint του **pool** (όχι στον τοπικό κόμβο)
   - Το pool χειρίζεται το `submit_nonce` προς την αλυσίδα

4. **Λειτουργία Pool**:
   - Το πορτοφόλι του pool έχει τα ιδιωτικά κλειδιά της διεύθυνσης σφυρηλάτησης
   - Το pool επικυρώνει τις υποβολές από τους miners
   - Το pool καλεί `submit_nonce` στο blockchain
   - Το pool διανέμει ανταμοιβές σύμφωνα με την πολιτική του pool

### Ανταμοιβές Coinbase

**Χωρίς Εκχώρηση**:
- Το coinbase πληρώνει απευθείας τη διεύθυνση του ιδιοκτήτη plot
- Ελέγξτε το υπόλοιπο στη διεύθυνση plot

**Με Εκχώρηση**:
- Το coinbase πληρώνει τη διεύθυνση σφυρηλάτησης
- Το pool λαμβάνει τις ανταμοιβές
- Ο miner λαμβάνει μερίδιο από το pool

**Πρόγραμμα Ανταμοιβών**:
- Αρχικά: 10 BTCX ανά block
- Υποδιπλασιασμός: Κάθε 1050000 blocks (~4 χρόνια)
- Πρόγραμμα: 10 → 5 → 2.5 → 1.25 → ...

## Αντιμετώπιση Προβλημάτων

### Συνήθη Ζητήματα

#### "Το πορτοφόλι δεν έχει ιδιωτικό κλειδί για τη διεύθυνση plot"

**Αιτία**: Το πορτοφόλι δεν κατέχει τη διεύθυνση
**Λύση**:
- Εισάγετε το ιδιωτικό κλειδί μέσω RPC `importprivkey`
- Ή χρησιμοποιήστε διαφορετική διεύθυνση plot που ανήκει στο πορτοφόλι

#### "Υπάρχει ήδη εκχώρηση για αυτό το plot"

**Αιτία**: Το plot είναι ήδη εκχωρημένο σε άλλη διεύθυνση
**Λύση**:
1. Ανακαλέστε την υπάρχουσα εκχώρηση
2. Περιμένετε την καθυστέρηση ανάκλησης (720 blocks testnet)
3. Δημιουργήστε νέα εκχώρηση

#### "Η μορφή διεύθυνσης δεν υποστηρίζεται"

**Αιτία**: Η διεύθυνση δεν είναι P2WPKH bech32
**Λύση**:
- Χρησιμοποιήστε διευθύνσεις που αρχίζουν με "pocx1q" (mainnet) ή "tpocx1q" (testnet)
- Δημιουργήστε νέα διεύθυνση αν χρειάζεται: `getnewaddress "" "bech32"`

#### "Το τέλος συναλλαγής είναι πολύ χαμηλό"

**Αιτία**: Συμφόρηση mempool δικτύου ή τέλος πολύ χαμηλό για αναμετάδοση
**Λύση**:
- Αυξήστε την παράμετρο ρυθμού τέλους
- Περιμένετε να αδειάσει το mempool

#### "Η εκχώρηση δεν είναι ακόμα ενεργή"

**Αιτία**: Η καθυστέρηση ενεργοποίησης δεν έχει παρέλθει ακόμα
**Λύση**:
- Ελέγξτε την κατάσταση: απομένοντα blocks μέχρι την ενεργοποίηση
- Περιμένετε να ολοκληρωθεί η περίοδος καθυστέρησης

#### "Δεν υπάρχουν διαθέσιμα κέρματα στη διεύθυνση plot"

**Αιτία**: Η διεύθυνση plot δεν έχει επιβεβαιωμένα UTXOs
**Λύση**:
1. Στείλτε κεφάλαια στη διεύθυνση plot
2. Περιμένετε 1 επιβεβαίωση
3. Δοκιμάστε ξανά τη δημιουργία εκχώρησης

#### "Δεν είναι δυνατή η δημιουργία συναλλαγών με πορτοφόλι μόνο παρακολούθησης"

**Αιτία**: Το πορτοφόλι εισήγαγε διεύθυνση χωρίς ιδιωτικό κλειδί
**Λύση**: Εισάγετε το πλήρες ιδιωτικό κλειδί, όχι μόνο τη διεύθυνση

#### "Η καρτέλα Εκχώρησης Σφυρηλάτησης δεν είναι ορατή"

**Αιτία**: Ο κόμβος ξεκίνησε χωρίς τη σημαία `-miningserver`
**Λύση**: Επανεκκινήστε με `bitcoin-qt -server -miningserver`

### Βήματα Αποσφαλμάτωσης

1. **Έλεγχος Κατάστασης Πορτοφολιού**:
   ```bash
   bitcoin-cli getwalletinfo
   ```

2. **Επαλήθευση Ιδιοκτησίας Διεύθυνσης**:
   ```bash
   bitcoin-cli getaddressinfo pocx1qplot...
   # Ελέγξτε: "iswatchonly": false, "ismine": true
   ```

3. **Έλεγχος Κατάστασης Εκχώρησης**:
   ```bash
   bitcoin-cli get_assignment pocx1qplot...
   ```

4. **Προβολή Πρόσφατων Συναλλαγών**:
   ```bash
   bitcoin-cli listtransactions "*" 10
   ```

5. **Έλεγχος Συγχρονισμού Κόμβου**:
   ```bash
   bitcoin-cli getblockchaininfo
   # Επαλήθευση: blocks == headers (πλήρως συγχρονισμένος)
   ```

## Βέλτιστες Πρακτικές Ασφαλείας

### Ασφάλεια Διεύθυνσης Plot

**Διαχείριση Κλειδιών**:
- Αποθηκεύστε τα ιδιωτικά κλειδιά διεύθυνσης plot με ασφάλεια
- Οι συναλλαγές εκχώρησης αποδεικνύουν την ιδιοκτησία μέσω υπογραφής
- Μόνο ο ιδιοκτήτης plot μπορεί να δημιουργήσει/ανακαλέσει εκχωρήσεις

**Αντίγραφα Ασφαλείας**:
- Δημιουργείτε τακτικά αντίγραφα ασφαλείας του πορτοφολιού (`dumpwallet` ή `backupwallet`)
- Αποθηκεύστε το wallet.dat σε ασφαλή τοποθεσία
- Καταγράψτε τις φράσεις ανάκτησης αν χρησιμοποιείτε HD πορτοφόλι

### Ανάθεση Διεύθυνσης Σφυρηλάτησης

**Μοντέλο Ασφαλείας**:
- Η διεύθυνση σφυρηλάτησης λαμβάνει ανταμοιβές block
- Η διεύθυνση σφυρηλάτησης μπορεί να υπογράφει blocks (εξόρυξη)
- Η διεύθυνση σφυρηλάτησης **δεν μπορεί** να τροποποιήσει ή να ανακαλέσει εκχώρηση
- Ο ιδιοκτήτης plot διατηρεί τον πλήρη έλεγχο

**Περιπτώσεις Χρήσης**:
- **Ανάθεση Θερμού Πορτοφολιού**: Κλειδί plot σε ψυχρή αποθήκευση, κλειδί σφυρηλάτησης σε θερμό πορτοφόλι για εξόρυξη
- **Εξόρυξη σε Pool**: Ανάθεση στο pool, διατήρηση ιδιοκτησίας plot
- **Κοινόχρηστη Υποδομή**: Πολλαπλοί miners, μία διεύθυνση σφυρηλάτησης

### Συγχρονισμός Ώρας Δικτύου

**Σημασία**:
- Η συναίνεση PoCX απαιτεί ακριβή ώρα
- Απόκλιση ρολογιού >10s ενεργοποιεί προειδοποίηση
- Απόκλιση ρολογιού >15s αποτρέπει την εξόρυξη

**Λύση**:
- Διατηρήστε το ρολόι συστήματος συγχρονισμένο με NTP
- Παρακολουθήστε: `bitcoin-cli getnetworkinfo` για προειδοποιήσεις απόκλισης χρόνου
- Χρησιμοποιήστε αξιόπιστους διακομιστές NTP

### Καθυστερήσεις Εκχωρήσεων

**Καθυστέρηση Ενεργοποίησης** (30 blocks testnet):
- Αποτρέπει ταχεία επανεκχώρηση κατά τη διάρκεια διχασμών αλυσίδας
- Επιτρέπει στο δίκτυο να επιτύχει συναίνεση
- Δεν μπορεί να παρακαμφθεί

**Καθυστέρηση Ανάκλησης** (720 blocks testnet):
- Παρέχει σταθερότητα για pools εξόρυξης
- Αποτρέπει επιθέσεις "griefing" εκχώρησης
- Η διεύθυνση σφυρηλάτησης παραμένει ενεργή κατά τη διάρκεια της καθυστέρησης

### Κρυπτογράφηση Πορτοφολιού

**Ενεργοποίηση Κρυπτογράφησης**:
```bash
bitcoin-cli encryptwallet "η_φράση_πρόσβασής_σας"
```

**Ξεκλείδωμα για Συναλλαγές**:
```bash
bitcoin-cli walletpassphrase "η_φράση_πρόσβασής_σας" 300
```

**Βέλτιστες Πρακτικές**:
- Χρησιμοποιήστε ισχυρή φράση πρόσβασης (20+ χαρακτήρες)
- Μην αποθηκεύετε τη φράση πρόσβασης σε απλό κείμενο
- Κλειδώνετε το πορτοφόλι μετά τη δημιουργία εκχωρήσεων
