## Об этой документации

Это полная техническая документация для Bitcoin-PoCX — интеграции Bitcoin Core с поддержкой консенсуса Proof of Capacity нового поколения (PoCX). Документация организована как навигируемое руководство со связанными главами, охватывающими все аспекты системы.

**Целевая аудитория**:
- **Операторы узлов**: Главы 1, 5, 6, 8
- **Майнеры**: Главы 2, 3, 7
- **Разработчики**: Все главы
- **Исследователи**: Главы 3, 4, 5

Переводы: [Арабский](../ara/index.md) [Французский](../fra/index.md) [Китайский](../zho/index.md) [Немецкий](../deu/index.md) [Японский](../jpn/index.md) [Корейский](../kor/index.md) [Испанский](../spa/index.md) [Вьетнамский](../vie/index.md)

## Содержание

### Часть I: Основы

**[Глава 1: Введение и обзор](1-introduction.md)**
Обзор проекта, архитектура, философия проектирования, ключевые особенности и отличия PoCX от Proof of Work.

**[Глава 2: Формат файлов графиков](2-plot-format.md)**
Полная спецификация формата графиков PoCX, включая оптимизацию SIMD, масштабирование доказательства работы и эволюцию формата от POC1/POC2.

**[Глава 3: Консенсус и майнинг](3-consensus-and-mining.md)**
Полная техническая спецификация механизма консенсуса PoCX: структура блока, сигнатуры генерации, корректировка базовой цели, процесс майнинга, конвейер валидации и алгоритм искривления времени.

### Часть II: Расширенные возможности

**[Глава 4: Система делегирования форджинга](4-forging-assignments.md)**
Архитектура на основе OP_RETURN для делегирования прав форджинга: структура транзакций, проектирование базы данных, конечный автомат, обработка реорганизаций и интерфейс RPC.

**[Глава 5: Синхронизация времени и безопасность](5-timing-security.md)**
Допуск отклонения часов, механизм защитного форджинга, противодействие манипуляциям с часами и вопросы безопасности, связанные с синхронизацией времени.

**[Глава 6: Сетевые параметры](6-network-parameters.md)**
Конфигурация chainparams, генезис-блок, параметры консенсуса, правила coinbase, динамическое масштабирование и экономическая модель.

### Часть III: Использование и интеграция

**[Глава 7: Справочник по RPC-интерфейсу](7-rpc-reference.md)**
Полный справочник по RPC-командам для майнинга, делегирования и запросов к блокчейну. Необходим для интеграции майнеров и пулов.

**[Глава 8: Руководство по кошельку и графическому интерфейсу](8-wallet-guide.md)**
Руководство пользователя для Qt-кошелька Bitcoin-PoCX: диалог делегирования форджинга, история транзакций, настройка майнинга и устранение неполадок.

## Сборка

### Сборка для разработки

```bash
# Клонирование с подмодулями
git clone --recursive https://github.com/PoC-Consortium/bitcoin-pocx.git
cd bitcoin-pocx/bitcoin

# Конфигурация с включённым PoCX
cmake -B build -DENABLE_POCX=ON

# Сборка
cmake --build build -j$(nproc)
```

**Варианты сборки**:
```bash
# С графическим интерфейсом Qt
cmake -B build -DENABLE_POCX=ON -DBUILD_GUI=ON

# Отладочная сборка
cmake -B build -DENABLE_POCX=ON -DCMAKE_BUILD_TYPE=Debug
```

**Зависимости**: Стандартные зависимости сборки Bitcoin Core. См. [документацию по сборке Bitcoin Core](https://github.com/bitcoin/bitcoin/tree/master/doc#building) для требований конкретных платформ.

### Релизные сборки

Для воспроизводимых релизных бинарных файлов используйте систему сборки GUIX: см. [bitcoin/contrib/guix/README.md](../bitcoin/contrib/guix/README.md)
